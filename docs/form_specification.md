# Спецификация формы подрядчика

## Назначение

Форма предназначена для подрядчиков — внесения данных о выполненных работах на объектах (корпусах).

**URL:** `/works/new`

**Доступ:** только пользователи с ролью `contractor`

## Поля формы

### 1. Номер корпуса (`building_id`)

- **Тип:** Выпадающий список (select)
- **Обязательное:** Да
- **Источник:** Справочник `buildings`
- **Валидация:** Значение должно существовать в справочнике

```
Опции:
  - 12 — Корпус 12
  - 15А — Корпус 15А
  - 20 — Корпус 20
```

### 2. Дата выполнения работы (`work_date`)

- **Тип:** Date picker
- **Обязательное:** Да
- **Формат:** DD.MM.YYYY
- **Валидация:** Не может быть в будущем

### 3. Вид работы (`service_id`)

- **Тип:** Выпадающий список с поиском (autocomplete / react-select)
- **Обязательное:** Да
- **Источник:** Справочник `services`
- **Поиск:** По подстроке в названии (не менее 2 символов)
- **Загрузка:** Динамическая (подгрузка совпадений с сервера)

**При выборе:**
- Подтягивается `unit` (единица измерения)
- Подтягивается `price` (цена за единицу)
- Цена отображается, но **не редактируется**

### 4. Количество (`service_quantity`)

- **Тип:** Числовое поле (input type="number", step="0.01")
- **Обязательное:** Да
- **Валидация:** > 0

**При изменении:**
- Автоматически пересчитывается сумма: `quantity × price`

### 5. Наименование работы (`description`)

- **Тип:** Текстовая область (textarea)
- **Обязательное:** Да
- **Валидация:** Минимум 5 символов, максимум 1000
- **Placeholder:** "Опишите выполненные работы"

### 6. Фотографии (`photos`)

- **Тип:** Drag & Drop зона + кнопка выбора
- **Обязательное:** Нет
- **Максимум:** 20 фото
- **Форматы:** JPG, JPEG, PNG
- **Максимальный размер оригинала:** 10 МБ на файл
- **Сжатие:** Автоматическое на сервере до 1 МБ (сохраняется качество 85%, размер до 1920px по длинной стороне)
- **Финальный размер:** ≤ 1 МБ на фото

**Отображение:**
- Превью загруженных фото (thumbnail)
- Кнопка удаления (❌) на каждом превью
- Прогресс загрузки для каждого файла

### 7. Файлы / Документы (`files`)

- **Тип:** Drag & Drop зона + кнопка выбора
- **Обязательное:** Нет
- **Форматы:** PDF, DOC, DOCX, XLS, XLSX
- **Максимальный размер:** 10 МБ на файл

**Отображение:**
- Список загруженных файлов с иконкой типа
- Кнопка удаления (❌)

### 8. Используемые материалы (`materials`)

- **Тип:** Множественный выбор с поиском (multi-select autocomplete)
- **Обязательное:** Нет (можно добавить позже)
- **Источник:** Справочник `materials`

**При добавлении материала:**
1. Открывается dropdown с поиском
2. Подрядчик вводит часть названия
3. Загружаются совпадения с сервера
4. При выборе материала:
   - Подтягивается `unit` и `price`
   - Появляется блок с полем "Количество"
   - Автоматически считается сумма

**Блок добавленного материала:**
```
┌────────────────────────────────┐
│ Название материала        [❌] │
│ Ед.изм.: Метр  Цена: 217.06 ₽  │
│ Количество: [  15.5  ]         │
│ Сумма: 3 364.43 ₽              │
└────────────────────────────────┘
```

**Валидация:**
- Количество > 0
- Материал не должен дублироваться

### 9. Итоговые суммы

Отображаются внизу формы и пересчитываются динамически:

| Поле | Формула |
|------|---------|
| Сумма по работам | `service_quantity × service_unit_price` |
| Сумма по материалам | `Σ (material_quantity × material_unit_price)` |
| **ИТОГО** | `service_total + materials_total` |

## Пользовательский сценарий

### Сценарий 1: Создание записи о работе

1. Подрядчик заходит по адресу `/works/new`
2. Выбирает номер корпуса из списка
3. Указывает дату выполнения работы
4. В поле "Вид работы" начинает вводить "монтаж"
5. Система предлагает варианты, подрядчик выбирает "Монтаж трубы"
6. Автоматически подтягивается единица измерения (м) и цена (350 ₽)
7. Подрядчик вводит количество: 15.5
8. Система пересчитывает сумму: 5 425 ₽
9. Вводит описание работы
10. Загружает 3 фото (drag & drop)
11. Добавляет материал "Труба ПП 20 мм", вводит количество 15.5
12. Добавляет материал "Фитинг угловой", вводит количество 3
13. Система пересчитывает итог: 9 057.93 ₽
14. Нажимает "Сохранить запись"
15. Система валидирует данные, сохраняет в БД, загружает файлы
16. Показывает сообщение об успехе и предлагает создать новую запись

### Сценарий 2: Ошибка валидации

1. Подрядчик заполняет форму, не указывает дату
2. Нажимает "Сохранить"
3. Система показывает ошибку: "Дата выполнения работы обязательна"
4. Поле подсвечивается красным, страница не перезагружается

## Валидация на фронтенде

| Поле | Правило | Сообщение об ошибке |
|------|---------|---------------------|
| Корпус | Обязательно | "Выберите корпус" |
| Дата | Обязательно, не в будущем | "Укажите корректную дату" |
| Вид работы | Обязательно | "Выберите вид работы" |
| Количество | > 0 | "Количество должно быть больше 0" |
| Описание | Мин. 5 символов | "Описание слишком короткое" |
| Фото | Макс. 20 шт., до 10 МБ | "Превышен лимит" |
| Материалы (кол-во) | > 0 | "Укажите количество" |

## Валидация на бэкенде

Все правила фронтенда дублируются на бэкенде + дополнительные:
- Проверка существования выбранного корпуса/услуги/материала
- Проверка прав доступа (роль contractor)
- Антивирусная проверка загружаемых файлов (опционально)

## UX / UI требования

1. **Автосохранение черновика** — данные формы сохраняются в `localStorage` каждые 30 секунд
2. **Подтверждение при уходе** — если форма заполнена и не сохранена, показывать предупреждение
3. **Мобильная адаптация** — форма должна корректно отображаться на экранах от 375px
4. **Загрузка файлов с прогрессом** — индикатор прогресса для каждого файла
5. **Быстрый поиск** — debounce 300ms при вводе в поля autocomplete
6. **Hot keys:**
   - `Ctrl+Enter` — сохранить форму
   - `Escape` — закрыть dropdown
