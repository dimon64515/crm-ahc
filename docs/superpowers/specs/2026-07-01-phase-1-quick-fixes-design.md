# Фаза 1 — Быстрые правки: спецификация

> **Goal:** Исправить печатную форму акта (добавить дату работ и русский формат дат), дать администратору возможность редактировать все поля записи о работе, и починить скачивание архива фотографий.

## Global Constraints

- Backend: Python 3.11+, FastAPI 0.111.0, SQLAlchemy 2.0.30, Pydantic 2.7.1.
- Frontend: React 19.2.6, Vite 8.0.12, Axios 1.16.1.
- База данных: PostgreSQL 15, миграции через Alembic.
- Все строки, сообщения об ошибках и комментарии — на русском языке.
- Ролевая модель: `contractor`, `director`, `admin`.
- Историчность цен: при создании работы цены фиксируются; при редактировании админом пересчёт выполняется по актуальным ценам справочника.
- Минимальные изменения существующей архитектуры; новые функции должны проходить через существующие роутеры `/api/works`, `/api/reports`, `/api/backups`.

---

## 1. Акт сдачи-приёмки (Word)

### Что меняем

- Добавить в таблицу работ столбец **«Дата работ»** между «№» и «Адрес объекта».
- Заменить английское название месяца в шапке документа (`today.strftime('%B')`) на русское через `_format_period_date`.
- Убедиться, что даты периода в пункте 2 акта уже используют `_format_period_date` (например, `«15» июня 2026 г.`).

### Файлы

- `backend/app/routers/reports.py` — функция `generate_act_docx`.

### Интерфейс

- Endpoint `/api/reports/act` не меняет сигнатуру параметров.
- Выходной документ `.docx` содержит новый столбец и русские даты.

## 2. Редактирование всех полей админом

### Что меняем

- Для роли `admin` в `PUT /works/{id}` разрешить передавать и изменять:
  - `building_id`
  - `service_id`
  - `work_date`
  - `service_quantity`
  - `description`
  - `user_id` (переназначение подрядчика)
  - `materials` — список объектов `{ material_id, quantity }`
- При смене `service_id` или `service_quantity`:
  - взять актуальную цену из `services.price`;
  - пересчитать `service_unit_price`, `service_total_price`, `total_price`.
- При изменении `materials`:
  - удалить существующие `WorkMaterial` для этой работы;
  - создать новые по актуальным ценам из `materials.price`;
  - пересчитать `materials_total_price` и `total_price`.
- Подрядчик (`contractor`) продолжает редактировать только `description`, `work_date`, `service_quantity`.
- Директор (`director`) остаётся без права редактирования.

### Файлы

- `backend/app/schemas.py` — расширить `WorkUpdate` полями для админа.
- `backend/app/routers/works.py` — логика `update_work` и вспомогательная функция пересчёта.

### Интерфейс

- `PUT /api/works/{id}`
  - Тело запроса (для admin): `WorkUpdateAdmin` = `WorkUpdate` + `building_id`, `service_id`, `user_id`, `materials: List[WorkMaterialCreate]`.
  - Ответ: `WorkResponse`.

## 3. Исправление скачивания архива фотографий

### Что меняем

- В `create_photos_backup` путь к фото формируется некорректно: `os.path.join(settings.UPLOAD_DIR, photo.file_path)`, но `photo.file_path` уже абсолютный.
- Исправить нормализацию пути:
  - если `photo.file_path` абсолютный — использовать его;
  - если относительный — присоединить к `UPLOAD_DIR`.
- Добавить проверку `os.path.exists(src)` перед копированием.
- В ответе возвращать реальное количество скопированных файлов.
- В `download_backup` убедиться, что `FileResponse` отдаёт файл с корректными заголовками.

### Файлы

- `backend/app/routers/backups.py` — функции `create_photos_backup` и `download_backup`.
- `backend/app/services/file_service.py` — уточнить, какой формат пути сохраняется в БД.

### Интерфейс

- `POST /api/backups/photos` — ответ: `{ success, backup_id, photos_count, size_mb, download_url }`.
- `GET /api/backups/download/{backup_id}` — возвращает ZIP-файл.

## Тестирование

- Ручное:
  - Сформировать акт Word и проверить наличие столбца «Дата работ» и русские даты.
  - Отредактировать работу админом: сменить корпус, вид работ, подрядчика, материалы; проверить пересчёт цен.
  - Создать архив фото и скачать его.
- Автоматизированное (опционально для Фазы 1):
  - `pytest` для `generate_act_docx` с проверкой содержимого DOCX.
  - `pytest` для `update_work` с ролью admin.
  - `pytest` для `create_photos_backup` с моками файловой системы.
